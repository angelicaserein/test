#### 代码测试

*U Help U* 的测试工作贯穿整个开发周期，分三个层次展开，分别针对不同的开发阶段和系统粒度。

**开发过程中的即时手动测试**

在整个开发过程中，每位成员在编写功能的同时进行持续的即时测试。每当新增或修改某项机制——例如调整录制状态机的转换逻辑、修改碰撞解算方式、或更新物理系统参数——开发者会立即在浏览器中运行游戏并手动验证行为是否符合预期。测试内容涵盖多种边界情况，包括同时按下多个按键、快速切换录制与回放状态、以及玩家与幻影在狭窄平台上的叠放交互等。问题在引入的第一时间被发现并修复，避免了错误向其他模块扩散。

**模块完成后的功能清单测试**

当某个系统模块达到稳定状态后，测试人员会依据结构化的功能测试清单对其进行系统性验证。每份清单针对一个具体系统——包括游戏流程、本地玩家系统与账号系统——按阶段组织测试用例，每条用例标注为正常路径、边界情况或潜在问题三类。测试人员在浏览器中手动逐项执行，标记通过或失败状态并记录观察到的异常。测试结果以交互式清单的形式保存，可通过以下链接查阅：

- [测试总览 — 所有模块](./docs/testing/测试总览.html)
- [游戏流程测试清单](./docs/testing/游戏流程测试清单.html)
- [本地玩家系统测试清单](./docs/testing/本地玩家系统测试清单.html)
- [账号系统测试清单](./docs/testing/账号系统测试清单.html)
